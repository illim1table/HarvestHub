# Sprint 5 输入 Prompt

> 📌 本文件归档于 `prompt/history/sprint5_next.md`。

---

你是一位资深的全栈开发工程师，正在与我共同开发"汇农亩场"农产品交易平台。我们已完成 **Sprint 4（交易下单与订单管理闭环 MVP）**，现在进入 **Sprint 5：支付回调模拟 + 订单状态流转完善 + 卖家订单管理**。

## 项目背景（延续 Sprint 1-4）

- **技术栈**：FastAPI（后端）+ Vue 3（前端）+ MySQL（数据库）
- **已完成能力（摘要）**：
  - 用户注册/登录（JWT）
  - 商品与分类持久化
  - 买家下单、订单列表/详情、取消待支付订单
  - 下单库存校验与扣减（后端金额计算）

## Sprint 5 任务目标

请完成以下内容：

### 1. 支付回调模拟（MVP）

- 增加模拟支付能力：
  - `POST /api/payments/mock`（或等价接口）用于模拟第三方支付回调。
- 回调参数至少包含：
  - `order_id`
  - `trade_no`（模拟交易号）
  - `pay_status`（如 `success` / `failed`）
- 回调需做幂等处理（同一订单重复 success 不应重复变更）。

### 2. 订单状态流转完善

- 明确并实现状态机约束（建议）：
  - `pending -> paid -> completed`
  - `pending -> cancelled`
  - 禁止非法流转（如 `cancelled -> paid`、`completed -> cancelled`）。
- 增加接口：
  - 买家确认收货：`PUT /api/orders/{id}/confirm`（`paid -> completed`）。
- 对状态变更增加必要时间字段（可新增）：
  - `paid_at`、`completed_at`、`cancelled_at`

### 3. 卖家订单管理（核心）

- 新增卖家视角订单能力：
  - `GET /api/seller/orders`：查看与自己商品相关的订单列表
  - `GET /api/seller/orders/{id}`：查看卖家订单详情（仅包含自己相关订单项，或按现有结构合理实现）
- 必须进行权限校验，禁止卖家越权查看无关订单。
- 支持按状态筛选（最小可用即可）。

### 4. 前端最小对接扩展

- 在“我的订单”中增加：
  - 待支付订单“去支付（模拟）”入口
  - 已支付订单“确认收货”入口
- 增加简易“卖家订单管理”页面/组件（卖家登录时可见）：
  - 展示订单与状态
  - 支持状态筛选与详情查看入口
- 对失败场景提供清晰提示（权限不足、非法状态流转、重复回调等）。

### 5. 可运行性与验证

- 提供可复现调试方式（Swagger/curl）覆盖：
  - 下单 -> 模拟支付成功 -> 买家确认收货 -> 卖家查看订单
- 保持现有项目结构、风格与可启动性。

## 输出要求

1. 提供所有新增/修改文件的完整代码，并在代码块上方注明文件路径。
2. 给出数据库迁移/初始化命令（含新增字段或表）。
3. 给出关键接口请求示例与完整联调步骤。
4. 保持 Prompt 路径规范：
   - 当前可执行 Prompt 放在 `prompt/sprintN_next.md`
   - 历史 Prompt 放在 `prompt/history/`
5. 在 Sprint 5 完成后，继续生成 `prompt/sprint6_next.md`，聚焦“退款/售后申请 + 简易消息通知 + 订单统计看板（卖家）”。
