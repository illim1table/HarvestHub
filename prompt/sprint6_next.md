你是一位资深的全栈开发工程师，正在与我共同开发"汇农亩场"农产品交易平台。我们已完成 **Sprint 5（支付回调模拟 + 订单状态流转完善 + 卖家订单管理）**，现在进入 **Sprint 6：退款/售后申请 + 简易消息通知 + 订单统计看板（卖家）**。

## 项目背景（延续 Sprint 1-5）

- **技术栈**：FastAPI（后端）+ Vue 3（前端）+ MySQL（数据库）
- **已完成能力（摘要）**：
  - 用户注册/登录（JWT）
  - 商品/分类持久化与管理
  - 买家下单、订单列表/详情、取消待支付订单
  - 模拟支付回调、买家确认收货
  - 卖家订单管理（列表/详情/筛选）

## Sprint 6 任务目标

请完成以下内容：

### 1. 退款/售后申请（MVP）

- 新增售后申请能力（建议订单级或订单项级最小可用实现）：
  - 买家发起售后申请（退款/退货退款）
  - 卖家处理售后申请（同意/拒绝）
- 建议接口：
  - `POST /api/aftersales`（买家发起）
  - `GET /api/aftersales`（按角色查看）
  - `PUT /api/aftersales/{id}/review`（卖家审核）
- 售后状态建议：`pending`、`approved`、`rejected`、`completed`。
- 必须有权限校验与状态机约束，禁止越权与非法流转。

### 2. 简易消息通知

- 新增最小消息通知能力（站内消息即可）：
  - 关键事件触发通知：
    - 买家发起售后
    - 卖家审核售后
    - 售后完成
- 建议接口：
  - `GET /api/notifications`（我的消息）
  - `PUT /api/notifications/{id}/read`（标记已读）
- 前端最小展示：显示未读/已读与时间。

### 3. 卖家订单统计看板

- 增加卖家统计接口与页面组件（MVP）：
  - 总订单数
  - 各状态订单数（pending/paid/completed/cancelled）
  - 已支付总金额（或已完成总金额，二选一并说明）
- 建议接口：
  - `GET /api/seller/dashboard/orders`
- 支持最小时间范围筛选（如最近7天/30天，或起止时间）。

### 4. 前端最小对接

- 买家端：售后申请入口与我的售后列表。
- 卖家端：售后处理入口、消息通知入口、订单统计看板。
- 对失败场景提供清晰提示（权限不足、非法流转、重复处理等）。

### 5. 可运行性与验证

- 提供可复现联调路径（Swagger/curl）覆盖：
  - 已支付订单 -> 发起售后 -> 卖家审核 -> 消息通知查看 -> 卖家看板统计变化
- 保持现有项目结构、风格与可启动性。

## 输出要求

1. 提供所有新增/修改文件的完整代码，并在代码块上方注明文件路径。
2. 给出数据库迁移/初始化命令（含新增字段或表）。
3. 给出关键接口请求示例与完整联调步骤。
4. 保持 Prompt 路径规范：
   - 当前可执行 Prompt 放在 `prompt/sprintN_next.md`
   - 历史 Prompt 放在 `prompt/history/`
5. Sprint 6 完成后，继续生成下一轮可执行 Prompt（Sprint 7）。
